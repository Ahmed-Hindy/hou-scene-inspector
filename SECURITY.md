# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for suspected secrets, license data,
proprietary files, or security vulnerabilities.

Use GitHub private vulnerability reporting if it is available for the
repository, or contact the maintainer privately at the email address listed in
`pyproject.toml`.

Include:

- A short description of the issue.
- The affected file, command, or fixture.
- Any steps needed to reproduce or verify the problem.
